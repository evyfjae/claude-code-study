설정 파일 작성

Ghostty 설정 파일 위치: ~/Library/Application Support/com.mitchellh.ghostty/config

key = value 형식, #으로 주석. Cmd+Shift+,로 재시작 없이 반영 가능.

~/Library/Application Support/com.mitchellh.ghostty/config 
# === 폰트 ===
font-family = JetBrains Mono
font-size = 14

# === 테마 (tmux Catppuccin Mocha와 통일) ===
theme = catppuccin-mocha

# === 창 설정 ===
window-padding-x = 4
window-padding-y = 4
window-decoration = true
macos-titlebar-style = tabs

# === macOS 전용 ===
macos-option-as-alt = true

# === tmux 자동 시작 ===
# Ghostty를 열면 바로 tmux main 세션에 연결
# -A: 있으면 attach, 없으면 새로 생성
command = /opt/homebrew/bin/tmux new-session -A -s main

# === 클립보드 ===
copy-on-select = clipboard

# === 커서 ===
cursor-style = block
cursor-style-blink = false

# === 기타 ===
confirm-close-surface = false
mouse-hide-while-typing = true
설정 옵션 상세 해설
theme = catppuccin-mocha
Ghostty에 내장된 Catppuccin Mocha 테마. tmux에서도 같은 테마를 쓰고 있으므로 색상이 통일됩니다. Ghostty는 터미널의 16색 팔레트를 설정하고, tmux Catppuccin은 상태바 등 UI 요소를 꾸밉니다 — 둘 다 설정해야 완전히 통일됩니다.
macos-option-as-alt = true
macOS에서 Option 키를 Alt/Meta로 사용합니다. tmux에서 M-key(Meta) 조합이나, vim의 Alt 단축키를 쓸 때 필수입니다. 이 설정 없이는 Option 키가 특수 문자(예: ∑, ¥)를 입력합니다.
command = /opt/homebrew/bin/tmux new-session -A -s main
Ghostty를 열면 셸 대신 이 명령어가 실행됩니다.
  • -A: attach-or-create. main 세션이 이미 있으면 연결, 없으면 새로 생성
  • -s main: 세션 이름을 main으로 지정
  • 전체 경로(/opt/homebrew/bin/tmux)를 쓰는 이유: Ghostty가 로그인 셸 프로파일(~/.zshrc)을 거치지 않고 직접 실행하기 때문에 PATH에 homebrew 경로가 없을 수 있음
macos-titlebar-style = tabs
네이티브 macOS 탭을 타이틀바에 통합합니다. 여러 Ghostty 탭을 깔끔하게 관리할 수 있습니다.
copy-on-select = clipboard
마우스로 텍스트를 선택하면 자동으로 클립보드에 복사됩니다. Cmd+C를 누를 필요가 없어 편리합니다.
confirm-close-surface = false
pane이나 탭을 닫을 때 확인 대화상자를 건너뜁니다. tmux 세션은 Ghostty를 닫아도 살아있으므로 안전합니다.
실습: 설정 적용하기
1
설정 파일 생성

Claude Code 세션에서 아래 명령어를 실행하세요:

mkdir -p ~/Library/Application\ Support/com.mitchellh.ghostty

그리고 위의 설정 내용을 config 파일로 저장하세요. Claude Code에게 "위 Ghostty 설정을 적용해줘"라고 하면 됩니다.

2
Ghostty 재시작 후 확인

Ghostty를 완전히 종료(Cmd+Q) 후 다시 열면, 자동으로 tmux main 세션에 진입해야 합니다.

3
tmux 안에서 Claude Code 실행

Ctrl+Space → %로 세로 분할 후, 한쪽에서 claude를 실행하세요. 다른 pane에서 자유롭게 작업하며 Claude Code와 동시에 사용해보세요.

이해 점검
Q. Ghostty를 종료했다가 다시 열면 Claude Code 세션은 어떻게 되는가?
Claude Code는 tmux 세션 안에서 실행 중이므로, Ghostty를 닫아도 tmux 세션은 백그라운드에서 계속 살아있습니다. Ghostty를 다시 열면 tmux new-session -A -s main이 기존 세션에 attach하므로, Claude Code 세션이 그대로 유지됩니다.
Q. -A 플래그 없이 tmux new-session -s main만 쓰면 어떤 문제가 생기는가?
이미 main 세션이 존재하면 "duplicate session: main" 에러가 발생하고 Ghostty가 빈 화면이 됩니다. -A가 있으면 기존 세션에 attach하므로 이 문제가 없습니다.
Q. Ghostty의 theme과 tmux의 Catppuccin 플러그인은 각각 무엇을 담당하는가?
Ghostty theme: 터미널의 기본 16색 팔레트(검정, 빨강, 초록 등)와 기본 전경/배경 색상을 설정합니다. 셸 프롬프트, ls 출력 색상 등에 영향을 줍니다.
tmux Catppuccin: tmux의 상태바, pane 테두리, 윈도우 탭 등 tmux UI 요소의 색상을 설정합니다.
둘 다 설정해야 터미널 전체가 통일된 Catppuccin Mocha 테마로 보입니다.

세션 자동 생성 스크립트

매번 수동으로 세션을 만들 필요 없이, 스크립트로 자동화할 수 있습니다.

~/bin/tmux-workspace.sh 
#!/bin/bash
# tmux 워크스페이스 자동 생성

# Study 세션
if ! tmux has-session -t study 2>/dev/null; then
  tmux new-session -d -s study -c ~/Documents/Projects/Claude\ Code\ Study
  tmux rename-window -t study:0 claude
  tmux split-window -h -t study:0 -c ~/Documents/Projects/Claude\ Code\ Study
  tmux new-window -t study -n docs
fi

# RUNuts 세션
if ! tmux has-session -t runuts 2>/dev/null; then
  tmux new-session -d -s runuts -c ~/Documents/Projects/RUNuts\ Project\ Managing
  tmux rename-window -t runuts:0 claude
  tmux split-window -h -t runuts:0 -c ~/Documents/Projects/RUNuts\ Project\ Managing
  tmux new-window -t runuts -n manage
fi

echo "워크스페이스 준비 완료!"
tmux ls

자동화 스크립트 적용

Claude Code에게 "위 워크스페이스 스크립트를 만들어줘"라고 요청하세요. 실행 후 tmux ls로 결과를 확인합니다.

이해 점검
Q. Session, Window, Pane의 차이를 한 줄씩 설명할 수 있는가?
Session: 독립적인 작업 환경 묶음. 프로젝트 단위로 분리. detach/attach의 단위.
Window: 세션 안의 탭. 작업 유형별로 구분. 번호로 전환.
Pane: 윈도우 안의 분할된 영역. 화면을 나누어 동시에 보기.
Q. 왜 프로젝트마다 별도의 세션을 만드는 것이 좋은가?
각 세션이 독립적인 working directory를 가지므로 프로젝트 간 파일 혼동이 없습니다. Claude Code를 프로젝트별로 실행하면 각각의 CLAUDE.md와 settings.json이 적용됩니다. 또한 세션별로 독립적으로 detach/attach할 수 있어, 한 프로젝트 작업을 잠시 멈추고 다른 프로젝트로 전환하기 쉽습니다.
