Phase 1 — 터미널 환경

1-1. Ghostty + tmux + Claude Code 기본 통합

터미널 환경을 완성하여 이후 모든 학습의 기반을 마련합니다.

왜 지금 이걸 배우는가?
  • 앞으로 32단계 학습이 전부 터미널에서 진행됨 — 환경이 불편하면 학습이 비효율적
  • Ghostty가 설치만 되어있고 설정 파일이 없는 상태 — 지금 바로 개선 가능
  • 파일 하나만 만들면 되니 부담이 낮고, 효과는 즉시 체감됨
먼저 알아야 할 기본 용어
터미널 (Terminal)
텍스트로 컴퓨터에 명령을 내리는 프로그램입니다. Mac에는 기본으로 "터미널" 앱이 있고, Ghostty는 더 빠르고 현대적인 대체 터미널 앱입니다. 쉽게 말해, 터미널 = 글자로 컴퓨터와 대화하는 창입니다.
tmux (Terminal Multiplexer)
하나의 터미널 창 안에서 여러 개의 화면을 동시에 관리하는 프로그램입니다. 비유하면, 터미널이 "책상"이라면 tmux는 책상 위에 "칸막이를 세워서 여러 작업 공간을 만드는 것"입니다. 가장 강력한 특징은 터미널을 닫아도 작업이 사라지지 않는다는 점입니다.
Prefix (프리픽스)
tmux에게 "지금부터 내가 하는 키 입력은 tmux 명령이야"라고 알려주는 시작 신호 키입니다. Jack님의 설정에서는 Ctrl+Space가 Prefix입니다.

예를 들어, 화면을 세로로 나누고 싶으면:
1. Ctrl+Space를 누른다 (Prefix — "tmux야, 명령 줄게")
2. 그다음 %를 누른다 ("화면을 세로로 나눠")

이것을 줄여서 "Prefix + %"라고 표기합니다. 두 키를 동시에 누르는 게 아니라, Prefix를 먼저 누르고 손을 뗀 후에 다음 키를 누릅니다.
Session (세션)
tmux에서의 독립적인 작업 환경 묶음입니다. 예를 들어, "study" 세션과 "runuts" 세션을 별도로 만들면 프로젝트별로 완전히 분리된 작업 공간을 가질 수 있습니다.
Pane (페인)
하나의 화면을 나눈 각각의 영역입니다. 예를 들어, 화면을 세로로 반 나누면 좌측 pane과 우측 pane이 생깁니다. 한쪽에서 Claude Code를 실행하고, 다른 쪽에서 결과를 확인하는 식으로 사용합니다.
Detach / Attach (분리 / 연결)
Detach: tmux 세션에서 "빠져나오기". 세션은 백그라운드에서 계속 살아있고, 나만 나온 것입니다. (Prefix + d)
Attach: 살아있는 세션에 다시 "들어가기". 빠져나왔던 세션에 다시 연결하면 이전 상태 그대로 보입니다. (tmux attach -t 세션이름)
세 가지 도구의 관계: 건물 비유
Ghostty (건물 외벽)
화면에 글자를 그리는 터미널 앱. 폰트, 색상, 창 크기를 결정합니다.
▼ 그 안에
tmux (건물 내부 칸막이)
터미널 안을 여러 작업 공간으로 나누고, 창을 닫아도 작업이 살아있게 합니다.
▼ 그 안에서
Claude Code (건물 안에서 일하는 AI)
tmux의 칸막이(pane) 안에서 실행되는 AI 코딩 도구입니다.
핵심 포인트: Ghostty를 닫아도 tmux는 백그라운드에서 살아있고, tmux 안의 Claude Code도 계속 실행 중입니다. 다시 Ghostty를 열면 이전 상태 그대로 돌아올 수 있습니다.

결론: tmux 분할을 주력으로 사용하세요. Ghostty는 "글자를 화면에 예쁘게 그리는 역할"에 집중하고, 작업 공간 관리는 tmux에 맡깁니다.
Ghostty 설정 파일 만들기

Ghostty의 설정 파일 위치: ~/Library/Application Support/com.mitchellh.ghostty/config

이 파일은 설정이름 = 값 형식으로, 한 줄에 하나의 설정을 작성합니다. #으로 시작하는 줄은 설명(주석)이며 무시됩니다.

Ghostty 설정 파일 전체 
# === 폰트 ===
font-family = JetBrains Mono
font-size = 14

# === 테마 (tmux Catppuccin Mocha와 통일) ===
theme = catppuccin-mocha

# === 창 설정 ===
window-padding-x = 4
window-padding-y = 4
window-decoration = true
macos-titlebar-style = tabs

# === macOS 전용 ===
# Option 키를 Alt로 사용 (tmux 단축키에 필요)
macos-option-as-alt = true

# === tmux 자동 시작 ===
# Ghostty를 열면 바로 tmux에 진입
# -A = 기존 세션이 있으면 연결, 없으면 새로 생성
command = /opt/homebrew/bin/tmux new-session -A -s main

# === 편의 설정 ===
copy-on-select = clipboard
cursor-style = block
cursor-style-blink = false
confirm-close-surface = false
mouse-hide-while-typing = true
주요 설정 하나씩 이해하기
theme = catppuccin-mocha
무엇? 터미널의 색상 테마입니다. Catppuccin Mocha는 눈이 편한 어두운 테마입니다.
왜? Jack님의 tmux에도 같은 Catppuccin 테마가 설정되어 있으므로, 색상이 통일됩니다.
macos-option-as-alt = true
무엇? Mac의 Option 키를 Alt 키로 사용하게 만듭니다.
왜? 이 설정 없이 Option을 누르면 ∑, ¥ 같은 특수 문자가 입력됩니다. tmux나 vim에서 Alt 조합 단축키를 쓰려면 이 설정이 필수입니다.
command = /opt/homebrew/bin/tmux new-session -A -s main
무엇? Ghostty를 열면 일반 셸(명령줄) 대신 이 명령어가 자동 실행됩니다.
의미 분해:
  • /opt/homebrew/bin/tmux — tmux 프로그램의 전체 경로. 왜 전체 경로? Ghostty가 시작될 때 PATH 환경변수가 아직 설정되지 않을 수 있어서 tmux만 쓰면 "명령어를 찾을 수 없음" 에러가 날 수 있음.
  • new-session — 새 세션을 만들어라
  • -A — 단, 이미 같은 이름의 세션이 있으면 새로 만들지 말고 기존 것에 연결해라 (Attach-if-exists)
  • -s main — 세션 이름을 "main"으로 해라
결과: Ghostty를 열 때마다 항상 tmux main 세션으로 들어갑니다.
copy-on-select = clipboard
무엇? 마우스로 텍스트를 드래그해서 선택하면 자동으로 클립보드에 복사됩니다.
왜? Cmd+C를 따로 누를 필요 없이, 선택만 하면 바로 다른 곳에 붙여넣기 가능.
confirm-close-surface = false
무엇? 탭이나 창을 닫을 때 "정말 닫으시겠습니까?" 확인을 건너뜁니다.
왜 안전한가? tmux 세션은 Ghostty를 닫아도 백그라운드에서 계속 살아있으므로, 실수로 닫아도 작업이 사라지지 않습니다.

3
tmux 기본 동작 테스트

tmux 안에서 아래를 순서대로 해보세요:

  • Ctrl+Space 누르고 → % 누르기 → 화면이 세로로 나뉘면 성공 (이것이 "Prefix + %" 동작)
  • Ctrl+Space 누르고 → 방향키 → 다른 pane으로 이동
  • 한쪽 pane에서 claude 입력 → Claude Code가 실행되면 성공
이해 점검
Q. Ghostty를 닫았다가 다시 열면 Claude Code가 사라지는가?
아닙니다. Claude Code는 tmux 세션 안에서 실행 중이므로, Ghostty를 닫아도 tmux 세션은 백그라운드에서 계속 살아있습니다. Ghostty를 다시 열면 -A 옵션 덕분에 기존 세션에 자동 연결되어, Claude Code가 그대로 보입니다.
Q. "Prefix + %"에서 Ctrl+Space와 %를 동시에 누르는 것인가?
아닙니다. 순서대로 누릅니다:
1. Ctrl+Space를 누르고 손을 뗀다 (tmux에게 "명령 줄게"라는 신호)
2. 그다음 %를 누른다 (실제 명령)
마치 자물쇠를 열고(Prefix) 문을 여는(명령 키) 것과 비슷합니다.
Q. Ghostty의 theme과 tmux의 Catppuccin은 왜 둘 다 설정해야 하는가?
Ghostty theme: 터미널 자체의 기본 색상(배경, 글자색, 오류 색상 등)을 설정합니다.
tmux Catppuccin: tmux의 상태바, 테두리, 탭 등 tmux UI만의 색상을 설정합니다.
하나만 설정하면 나머지 영역이 다른 색으로 보여서 어색합니다. 둘 다 같은 테마로 맞추면 화면 전체가 통일됩니다.

tmux의 3계층 구조: 회사 비유

tmux의 구조를 회사 사무실에 비유하면 쉽게 이해할 수 있습니다.

Session (세션) = 사무실 층
가장 큰 단위. 프로젝트별로 하나씩 만듭니다.
예: "study" 세션 = 학습 전용 층, "runuts" 세션 = RUNuts 프로젝트 층
명령어: tmux new -s study (study라는 이름의 새 세션 생성)
Window (윈도우) = 사무실 방
세션 안의 탭 같은 개념. 용도별로 나눕니다.
예: Window 0 = Claude Code 방, Window 1 = 문서 작업 방
만들기: Prefix + c  |  전환: Prefix + 0, Prefix + 1 (번호로)
이름 바꾸기: Prefix + ,
Pane (페인) = 방 안의 책상
하나의 윈도우를 분할한 영역. 동시에 여러 것을 볼 때 사용.
예: 좌측 pane = Claude Code, 우측 pane = 파일 목록
세로 분할: Prefix + %  |  가로 분할: Prefix + "
pane 간 이동: Prefix + 방향키
자주 쓰는 tmux 명령어 정리
기억하세요: 아래에서 "Prefix"는 모두 Ctrl+Space를 먼저 누르는 것을 의미합니다. "Prefix + d"는 Ctrl+Space 누르고 손 떼고 → d 누르기입니다.
세션 관리
tmux new -s 이름 — 새 세션 만들기
tmux ls — 현재 세션 목록 보기
tmux attach -t 이름 — 세션에 다시 들어가기
Prefix + d — 세션에서 나오기 (세션은 계속 살아있음)
Prefix + s — 세션 목록을 띄워서 선택하며 전환
윈도우 관리
Prefix + c — 새 윈도우(탭) 만들기
Prefix + 0~9 — 번호로 윈도우 전환
Prefix + , — 현재 윈도우 이름 바꾸기
Pane 관리
Prefix + % — 세로로 나누기 (좌 | 우)
Prefix + " — 가로로 나누기 (상 / 하)
Prefix + 방향키 — pane 간 이동
Prefix + z — 현재 pane 전체화면 확대/복원 (줌)
Prefix + x — 현재 pane 닫기
추천 워크스페이스 구조

아래와 같이 프로젝트별로 세션을 분리하면, 작업 간 혼동이 없고 전환이 빠릅니다.

워크스페이스 레이아웃
Session: main (기본 세션 — Ghostty를 열면 여기로 진입)
  └── 일회성 작업, 시스템 관리

Session: study (Claude Code 학습 전용)
  ├── Window 0: claude
  │     ├── 좌측 Pane: Claude Code 실행
  │     └── 우측 Pane: 명령어 테스트, git 확인
  └── Window 1: docs — 문서 편집, 참고

Session: runuts (RUNuts 프로젝트 전용)
  ├── Window 0: claude
  │     ├── 좌측 Pane: Claude Code 실행
  │     └── 우측 Pane: 프로젝트 작업
  └── Window 1: manage — git, 파일 탐색
세션 전환 방법: Prefix + s를 누르면 세션 목록이 나옵니다. 방향키로 선택하고 Enter로 전환합니다. 마치 Alt+Tab으로 앱을 전환하듯, 프로젝트 간에 빠르게 이동할 수 있습니다.
세션 자동 생성 스크립트

매번 세션을 수동으로 만들 필요 없이, 스크립트 하나로 전체 환경을 자동 구성할 수 있습니다. 이 스크립트의 의미를 한 줄씩 설명합니다.

~/bin/tmux-workspace.sh 
#!/bin/bash
# tmux 워크스페이스 자동 생성 스크립트

# --- Study 세션 ---
# "study 세션이 이미 있는가?" 를 확인하고, 없으면 만든다
if ! tmux has-session -t study 2>/dev/null; then
  # 새 세션 생성 (-d: 백그라운드로, -s: 이름, -c: 시작 디렉토리)
  tmux new-session -d -s study -c ~/Documents/Projects/Claude\ Code\ Study
  # 첫 번째 윈도우 이름을 "claude"로 변경
  tmux rename-window -t study:0 claude
  # 세로 분할 (-h: horizontal split = 좌|우)
  tmux split-window -h -t study:0 -c ~/Documents/Projects/Claude\ Code\ Study
  # 두 번째 윈도우 "docs" 추가
  tmux new-window -t study -n docs
fi

# --- RUNuts 세션 ---
if ! tmux has-session -t runuts 2>/dev/null; then
  tmux new-session -d -s runuts -c ~/Documents/Projects/RUNuts\ Project\ Managing
  tmux rename-window -t runuts:0 claude
  tmux split-window -h -t runuts:0 -c ~/Documents/Projects/RUNuts\ Project\ Managing
  tmux new-window -t runuts -n manage
fi

echo "워크스페이스 준비 완료!"
tmux ls  # 만들어진 세션 목록을 보여줌
실습
1
세션 직접 만들어보기

터미널에서 tmux new -s study를 입력하여 study 세션을 만들어보세요. 하단 상태바에 "study"가 표시되면 성공입니다.

2
Detach와 Attach 체험

Prefix + d (= Ctrl+Space 후 d)로 세션에서 나오세요. 그 후 tmux ls로 세션이 살아있는지 확인하고, tmux attach -t study로 다시 들어가보세요.

3
자동화 스크립트 적용

Claude Code에게 "위 워크스페이스 스크립트를 만들고 실행해줘"라고 요청하세요. tmux ls로 study, runuts 세션이 모두 생성되었는지 확인.

이해 점검
Q. Session, Window, Pane을 한 줄씩 설명해보세요.
Session(세션): 가장 큰 단위. 프로젝트별로 하나씩. 닫아도 백그라운드에서 유지됨. (회사 비유: 사무실 층)
Window(윈도우): 세션 안의 탭. 용도별로 나눔. 번호로 전환. (회사 비유: 사무실 방)
Pane(페인): 윈도우 안에서 화면을 나눈 영역. 동시에 여러 것을 볼 때 사용. (회사 비유: 방 안의 책상)
Q. Prefix + d로 세션에서 나오면, 그 안에서 실행 중이던 프로그램은 어떻게 되는가?
계속 실행됩니다. Detach는 "화면 연결을 끊는 것"이지, 프로그램을 종료하는 것이 아닙니다. Claude Code가 긴 작업을 하고 있었다면, detach해도 백그라운드에서 계속 작업합니다. 나중에 tmux attach로 다시 들어가면 작업 결과를 그대로 볼 수 있습니다.
Q. 왜 프로젝트마다 별도의 세션을 만드는 것이 좋은가?
세 가지 이유입니다:
1. 파일 분리: 각 세션이 자기만의 작업 디렉토리를 가지므로 프로젝트 간 파일이 섞이지 않음
2. Claude Code 분리: 프로젝트별로 Claude Code를 실행하면 각각의 CLAUDE.md 규칙이 적용됨
3. 독립적 관리: 한 프로젝트를 잠시 놔두고(detach) 다른 프로젝트로 전환하기 쉬움
